# 文档同步检查 Skill

## 概述

在开发过程中，代码变更往往需要同步更新多个相关文档。本 Skill 定义了**功能开发完成后的文档同步检查流程**，确保代码、文档、配置保持一致。

## 触发时机

以下情况必须执行本文档同步检查：

1. 新增/修改/删除功能模块
2. 新增/修改/删除 API 接口
3. 新增/修改/删除菜单/路由
4. 修改数据库 Schema
5. 修改文件结构
6. 完成功能清单中的任务项

## 同步检查清单

### 1. 菜单变更检查

**触发条件**：新增了菜单模块、修改了菜单结构

**需要更新的文件**：
- [ ] `backend/prisma/seed.ts` - 添加/更新菜单种子数据
- [ ] `AGENTS.md` - 更新项目结构中的菜单说明

**检查要点**：
```typescript
// seed.ts 中检查
1. 菜单 ID 是否唯一（避免与现有菜单冲突）
2. parentId 是否正确设置
3. 菜单是否分配给 admin 角色
4. 菜单路径与前端路由是否一致
```

---

### 2. 功能模块完成检查

**触发条件**：完成了开发计划中的某个功能模块

**需要更新的文件**：
- [ ] `version/v*.md` - 更新对应版本的功能清单状态

**检查要点**：
```markdown
- [x] 已完成的任务改为勾选状态
- 添加完成日期标记
- 如有变更，更新功能描述
```

---

### 3. API 变更检查

**触发条件**：新增/修改/删除 API 接口

**需要更新的文件**：
- [ ] `version/v*.md` - 更新 API 设计文档
- [ ] `AGENTS.md` - 更新 API 架构说明
- [ ] `.claude/rules/api-design.md` - 如有新的 API 模式

**检查要点**：
```
1. RESTful 规范是否符合
2. 接口路径是否正确记录
3. 请求/响应参数是否更新
```

---

### 4. 数据库 Schema 变更检查

**触发条件**：修改了 `schema.prisma`

**需要更新的文件**：
- [ ] `backend/prisma/seed.ts` - 添加种子数据（如需要）
- [ ] `version/v*.md` - 更新数据库设计文档
- [ ] `AGENTS.md` - 更新数据库架构说明

**检查要点**：
```
1. 是否需要初始化数据（如单位、分类等）
2. 字段说明是否更新
3. 关系定义是否正确
```

---

### 5. 文件结构变更检查

**触发条件**：新增/删除/重命名目录或重要文件

**需要更新的文件**：
- [ ] `AGENTS.md` - 更新项目结构章节

**检查要点**：
```
1. 目录树是否准确
2. 新增模块是否在结构图中体现
3. 文件说明是否准确
```

---

### 6. 前端路由变更检查

**触发条件**：新增/修改前端页面路由

**需要更新的文件**：
- [ ] `frontend/src/router/index.ts` - 确保路由已添加
- [ ] `AGENTS.md` - 更新前端结构说明

## 执行流程

### Step 1: 识别变更类型

分析本次修改涉及哪些方面：

```
□ 数据库 Schema 变更
□ 后端模块新增/修改
□ API 接口变更
□ 前端页面新增/修改
□ 菜单/路由变更
□ 配置文件变更
```

### Step 2: 执行同步检查

根据变更类型，执行对应的检查清单：

```
如果 Schema 变更 → 执行检查清单 #4
如果新增菜单 → 执行检查清单 #1、#5
如果完成功能 → 执行检查清单 #2
...
```

### Step 3: 更新文档

按照优先级依次更新：

1. **高优先级**：seed.ts、配置文件（影响运行）
2. **中优先级**：version/*.md（进度跟踪）
3. **低优先级**：AGENTS.md（项目说明）

### Step 4: 验证同步

确认以下一致性：

```
□ 代码 ↔ seed.ts（菜单、初始数据）
□ 代码 ↔ version/*.md（功能清单、API）
□ 代码 ↔ AGENTS.md（项目结构）
```

## 常见问题

### Q: 如何确定菜单 ID？

查看现有菜单最大 ID，新菜单使用更大 ID（间隔 10，方便插入）：

```typescript
// 现有菜单 ID: 1, 2, 3, 4
// 新菜单从 10 开始：10, 11, 12...
```

### Q: 功能清单如何标记完成？

```markdown
修改前：
- [ ] 计量单位管理

修改后：
- [x] 计量单位管理 ✓ 2026-02-11
```

### Q: 如何保持 AGENTS.md 结构清晰？

使用统一的层级格式：

```markdown
backend/src/
├── units/           # 计量单位模块（新增）
├── categories/      # 商品分类模块（新增）
└── ...
```

## 总结

**核心原则**：
1. **一次变更，多处同步** - 代码修改后必须检查相关文档
2. **高优先级优先** - 先更新影响运行的配置（seed.ts）
3. **及时更新** - 功能完成后立即更新，不要积压
4. **验证一致性** - 确保代码与文档描述一致

**记忆口诀**：
> 改代码，想同步  
> 菜单变，改 seed  
> 功能完，勾清单  
> 结构变，更 AGENTS
