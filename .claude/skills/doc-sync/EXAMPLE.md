# 文档同步检查示例

> 本文档演示如何在完成基础数据模块后，执行文档同步检查

---

## 场景

完成了 **1.1 基础数据模块** 的开发，涉及：
- ✅ 后端：6 个 NestJS 模块（units, categories, brands, warehouses, suppliers, customers）
- ✅ 前端：6 个 Vue 页面 + API + 类型定义
- ✅ 数据库：6 张新表
- ✅ 路由：新增 6 个路由
- ✅ 菜单：新增 7 个菜单（1 个父菜单 + 6 个子菜单）

---

## Step 1: 识别变更类型

根据本次修改，涉及以下方面：

```
☑ 数据库 Schema 变更 - 新增 6 张表
☑ 后端模块新增 - 新增 6 个模块
☑ API 接口变更 - 新增 6 组 REST API
☑ 前端页面新增 - 新增 6 个页面
☑ 菜单/路由变更 - 新增 7 个菜单、6 个路由
☑ 功能完成 - 1.1 基础数据模块完成
```

## 如果没有执行同步检查会发生什么？

### 情况 1: seed.ts 未更新菜单

```
问题：新用户部署系统后，侧边栏看不到"基础数据"菜单
影响：无法通过 UI 访问基础数据模块
临时解决：手动访问 URL (/basic/units)
```

### 情况 2: version/v1.0.md 未更新

```
问题：项目进度不透明，不知道哪些功能已完成
影响：团队协作困难，可能重复开发
```

### 情况 3: AGENTS.md 未更新

```
问题：新开发者查看 AGENTS.md 时，看不到基础数据模块
影响：项目结构不清晰，上手困难
```

---

## 总结

通过执行文档同步检查，确保：

1. **代码与种子数据一致** - 新用户部署即可使用
2. **代码与进度文档一致** - 项目状态清晰可见
3. **代码与架构文档一致** - 项目结构准确描述

**核心原则：一次代码变更，所有相关文档同步更新**
